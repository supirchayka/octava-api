# Разделение категорий услуг по полу и новые ручки

## Что изменилось
- Добавлено поле `gender` для категорий услуг, чтобы разделять мужские и женские направления.
- В публичных ответах категорий и услуг теперь возвращается признак пола категории.
- Появились отдельные ручки для получения женских и мужских категорий, а также полного списка услуг по полу.

## Пол категории
- Возможные значения: `FEMALE` и `MALE`.
- Если при создании категории пол не указан, используется значение по умолчанию `FEMALE`.

## Новые публичные ручки

### Категории по полу
- `GET /service-categories/female` — список женских категорий услуг.
- `GET /service-categories/male` — список мужских категорий услуг.

Ответ совпадает с `GET /service-categories`, дополнительно возвращается поле `gender`.

### Услуги по полу
- `GET /services/female` — список женских услуг (по полу категории).
- `GET /services/male` — список мужских услуг (по полу категории).

Каждый элемент услуги содержит блок `category` с `id`, `slug`, `name`, `gender`.

## Что учесть при администрировании
- В админских запросах на создание/обновление категории (`/admin/catalog/categories`) добавлено поле `gender`.
- Новое поле хранится в категории и используется для фильтрации в новых публичных ручках.
